regPoisson_ozone

Régression de Poisson : prédiction de l’ozone

Introduction

Contexte et justification

Le réchauffement climatique est aujourd’hui l’un des plus grands problèmes auxquels fait face la planète Terre, et est la juste conséquence de la mauvaise exploitation des ressources par l’homme. Et l’un des effets les plus critiques est la destruction de la couche d’ozone responsable de l’effet de serre. Pour lutter contre un tel phénomène, il est important de comprendre l’évolution des différentes entités environnementales en jeu, et l’ozone en est un parfait exemple. Comprendre comment modéliser les transformations qu’elle subit en fonction de plusieurs facteurs est impératif si l’on veut prendre des mesures radicales et avisées. Ce travail consistera donc à modéliser le niveau de l’ozone, en se basant sur le dataset airquality qui mesure la qualité de l’air à New York, de façon journalière, de mai à septempbre 1973.

Choix des données

Le niveau d’ozone est en général mesuré en ppb ou parts per billion et est une unité de mesure de concentration représentant une partie de soluté pour un milliard \(10^9\) de parties de solution. Utilisé pour de très faibles concentrations (contaminants, air, eau), \(1 \text{ ppb} = 1 \mu g/L \text{ ou } 1 \mu g / kg\). Par définition donc, c’est une valeur qui doit être strictement non-négative. De plus, les analyseurs d’ozone (photométrie UV) ont en général une résolution de 1 pb, ce qui en fait des mesures toutes en nombres entiers. Ces conditions sont toutes respectées dans le dataset airquality et constituent les raisons pour lesquelles nous avons choisi ces données.

Modèle de Poisson

Le modèle de Poisson est réalisé grâce à la fonction glm() avec l’argument family = "poisson".

L’expression du modèle est donc : \[\ln{\lambda_{Ozone}} = \beta_0 + \beta_1 \cdot \text{Solar.R} + \beta_2 \cdot \text{Wind} + \beta_3 \cdot \text{Temp}\]

avec \(\beta_0 = 1.457\), \(\beta_1 = 0.0025\), \(\beta_2 = -1.004\), \(\beta_3 = 0.0334\) .

Comme on peut le remarquer, tous les coefficients du modèle sont tous très significatifs.

Interprétation des coefficients

On dira alors qu’une augmentation d’une unité de Solar.R (rayonnement solaire) entraîne une augmentation de 0.0025 ppb de la concentration moyenne d’Ozone. La même logique est utilisée pour les autres coefficients.

Une meilleure interprétation se fait en calculant les ratios de taux d’incidence (IRR) via \(\exp(\hat{\beta}_j)\) pour chaque coefficient. Ainsi, on a :

  • Solar.R : exp(0.0025131) = 1.002516. Une augmentation d’une unité du rayonnement solaire augmente la concentration d’ozone de 0.25%, toutes choses étant égales par ailleurs

  • Wind : exp(-0.1003999) = 0.9044756. Pour chaque augmentation de 1 unité de la vitesse du vent, la concentration attendue d’ozone diminue de 9,55% (1-0.904).

  • Temp : exp(0.0334104) = 1.033975. Pour chaque augmentation de 1 degré de température, la concentration attendue d’ozone augmente de 3,4%.

Validation du modèle

Vérification des hypothèses

  1. Indépendance des réponses (Ozone)

    On considère en général que cette hypothèse est satisfaite.

  2. Distribution de Poisson pour les réponses (Ozone)

    Pour cela, on va comparer la distribution des valeurs observées avec la distribution théorique de Poisson de paramètre la moyenne observée.

lambda_obs <- mean(trainCleaned$Ozone)

# simulation des comptages
set.seed(123)
theoretic_oz <- rpois(length(trainCleaned$Ozone), lambda = lambda_obs)

# affichage simultané
ggplot(trainCleaned) + 
  geom_bar(aes(trainCleaned$Ozone, fill = "#0000FF")) +
  geom_bar(aes(theoretic_oz, fill = "#FF2100")) +
  ggtitle("Ozone vs Distribution de Poisson") + 
  scale_fill_discrete(name = "Légende", labels = c("Ozone", "Poisson théorique"))

On peut voir que les deux distributions sont assez différentes. La variable Ozone ne suit donc pas une distribution de poisson de paramètre 42.78. De plus, les valeurs sont très dispersées sur l’axe des abscisses, ce qui peut être un effet de la surdispersion.

  1. Surdispersion

    La surdispersion a déjà été remarquée précédemment, donc nous allons nous contenter de faire un test pour la confirmer, celui du Z-Score.

    \[ Z_i = \frac{(y_i - \lambda_i)^2 - y_i}{\lambda_i \sqrt{2}} \sim t_{1,\alpha} \]

z <- ((trainCleaned$Ozone - lamda)^2 - trainCleaned$Ozone) / (lamda * sqrt(2))
zscore <- lm(z ~ 1)

    weights <- weights / mean(weights) # normalisation
    return(weights)
}

Qualité et Pertinence

Jusque-là, nous avons 6 modèles : poisson normal et avec poids, quasi-poisson avec poids et binomial négatif, et avec poids.

Pour analyser la pertinence de chacun de ces modèles, nous allons faire les tests de :

  • Test de Hosmer-Lemeshow

  • Test des résidus de Pearson

Quant à la qualité, nous allons utiliser

  • le score R²

  • la précision des prédiction

  • les mesures d’erreurs : RMSE, MSE et RSE etc

#' Comparaison graphique des prédictions d'un modèle (entraînement vs test)
#'
#' @param model Un objet modèle R (ex: lm, glm) contenant les valeurs ajustées.
#' @param test_data Un data.frame ou une matrice contenant les prédicteurs du test.
#' @param test_response Vecteur des valeurs réelles de la variable réponse pour le test.
#' @param test_only Booléen. Si TRUE, seul le jeu de test est affiché.
#'
#' @return Un objet ggplot représentant les valeurs observées vs prédites.
#' @export
#'
#' @examples
#' modele <- lm(mpg ~ wt + hp, data = mtcars[1:20, ])
#' p <- plot_pred_accuracy(modele, mtcars[21:32, ], mtcars$mpg[21:32])
#' print(p)
plot_pred_accuracy <- function(model, test_data, test_response, 
                               test_only = FALSE,
                               plot_title = NULL,
                               plot_subtitle = NULL) {
  
  if (!requireNamespace("ggplot2", quietly = TRUE))
    stop("Le package ggplot2 est requis. Installez-le avec install.packages('ggplot2').")
  
  # Prédictions entraînement (déjà sur l'échelle de la réponse pour un GLM)
  train_pred <- stats::fitted(model)
  train_obs  <- stats::model.frame(model)[, 1]
  
  # Prédictions test (échelle de la réponse)
  test_pred <- stats::predict(model, newdata = test_data, type = "response")
  
  # Vérification rapide
  if (stats::var(test_pred) == 0) {
    warning("Les prédictions sur le test sont constantes (variance nulle). Vérifiez le modèle et les données.")
  }
  
  # Construction des données pour ggplot
  if (test_only) {
    plot_points <- data.frame(Outcome = test_response, Prediction = test_pred, 
                              DataSet = "test")
    plot_residus <- data.frame(x = test_response, y = test_pred,
                               xend = test_response, yend = test_response, 
                               DataSet = "test")
  } else {
    df_train <- data.frame(Outcome = train_obs, Prediction = train_pred, 
                           DataSet = "entraînement")
    df_test  <- data.frame(Outcome = test_response, Prediction = test_pred, 
                           DataSet = "test")
    plot_points <- rbind(df_train, df_test)
    
    res_train <- train_obs - train_pred
    res_test  <- test_response - test_pred
    df_train_res <- data.frame(x = train_obs, y = train_pred,
                               xend = train_obs, yend = train_obs, 
                               DataSet = "entraînement")
    df_test_res  <- data.frame(x = test_response, y = test_pred,
                               xend = test_response, yend = test_response, 
                               DataSet = "test")
    plot_residus <- rbind(df_train_res, df_test_res)
  }
  
  r2_test <- round(stats::cor(test_pred, test_response)^2, 3)
  
  # Titres par défaut si non fournis
  if (is.null(plot_title)) {
    plot_title <- "Observé vs Prédit"
  }
  if (is.null(plot_subtitle)) {
    plot_subtitle <- if (test_only) "Jeu de test uniquement" else "Jeux d'entraînement et de test"
  }
  
  p <- ggplot2::ggplot() +
    ggplot2::geom_abline(slope = 1, intercept = 0, color = "red", linetype = "dashed") +
    ggplot2::geom_point(data = plot_points,
                        ggplot2::aes(x = Outcome, y = Prediction, color = DataSet),
                        size = 2.5, alpha = 0.8) +
    ggplot2::geom_segment(data = plot_residus,
                          ggplot2::aes(x = x, y = y, xend = xend, yend = yend, color = DataSet),
                          alpha = 0.3) +
    ggplot2::labs(x = "Valeurs observées", y = "Valeurs prédites",
                  title = plot_title,
                  subtitle = plot_subtitle,
                  color = "Jeu de données") +
    ggplot2::theme_minimal() +
    ggplot2::theme(legend.position = "bottom")
  
  # Annotation du R²
  max_val <- max(plot_points$Outcome, plot_points$Prediction)
  p + ggplot2::annotate("text", x = 0.05 * max_val, y = 0.95 * max_val,
                        label = paste0("R² test : ", r2_test),
                        size = 5, hjust = 0, vjust = 1, fontface = "bold")
}
